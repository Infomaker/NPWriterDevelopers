    • Newspilot Writer

    General information

    This is a node server responsible for serving the Newspilot Writer web application. It also routes communication with Editor Service and Concept Backend (optional). The service is running in a docker container on several EC2 instances.

    System architecture

    Operations information

    Newspilot Writer executes in a docker container in AWS. The docker containers are administrated by AWS EC2 Container Service.

    Logs

    The logs can be found in two places, cloudwatch or in files on the machines running the instances.

    $ ssh ec2-user@instance -i ~/path/to/pemfile
instance$ tail /var/log/writer/writer.log

    Configuration

    The service is configured in environment variables in AWS EC2 Container Service. The task definition dictates which image to run together with the configuration. Below is a list of possible configuration entries:

    Key Description
    AWS_ACCESS_KEY_ID Access key used for reading config file
    AWS_SECRET_ACCESS_KEY Accompanying secret key
    AWS_S3_CLIENT_CONFIG_NAME Configuration file for clients connecting to writer server
    AWS_S3_SERVER_CONFIG_NAME Configuration file for writer server
    AWS_S3_BUCKET_NAME S3 bucket containing the config files
    LOG_PATH Path to log files
    LOG_LEVEL debug, info, warning or error
    LOG_FILES Number of log files to keep
    NODE_ENV production when running on AWS, development if running locally and standby for not configured cloud formation setup
    FORCE_SECURE_CONNECTIONS If true, http requests towards Newspilot Writer will be redirected to https. Default to http

    Development configuration

    The environment variables below can be used to tweak the writer backend during development

    Key Description
    IN_DOCKER true to configure service for running inside docker, false otherwise
    CONFIG_FILE The name of the client configuration file used during development. Default is writer.json
    SERVER_CONFIG_FILE The name of the server configuration file used during development. Default is server.json

    New relic monitoring

    Key Description
    NEWRELIC_ENABLED Optional. True to enable monitoring
    NEWRELIC_APP_NAME Optional. Name of application to appear in new relic
    NEWRELIC_LOG_LEVEL Optional. Default 'info'
    NEWRELIC_LICENSE Optional. New relic license

    Configuration files

    The writer server uses 2 configuration files.

    • The AWS_S3_SERVER_CONFIG_NAME file is used to configure the server. This is read once during boot.
    • The AWS_S3_CLIENT_CONFIG_NAME specifies how clients connecting to the server should be configured. This file is monitored periodically by the running server, and when changes are detected, the file is being re-read.

    Server config file

    {
  "server": {
    "port": 5000
  },
  "external": {
    "contentrepository": {
      "protocol": "https:",
      "host": "46ihx4v1i9.execute-api.eu-west-1.amazonaws.com",
      "endpoint": "dev/cxf/writer/v2/commands",
      "apiKey": "[enter api key here]"
    },
    "conceptbackend": {
      "protocol": "http:",
      "host_old": "52.212.111.16",
      "host": "52.212.123.122",
      "port": "3000",
      "endpoint": "commands",
      "apiKey": null
    }
  },
  "debugProxyCalls": false,
  "binary": {
    "maxSizeMb": {
      "x-im/image": 10,
      "x-im/pdf": 10
    },
    "binaryHeadRequestTimeoutMs": 500
  }
}
    • server.port - Specifies which port the writer server should listen to
    • external.contentrepository - Configuration details for how to connect to Editor Service backend
    • external.contentrepository.apiKey - This is required for connecting to Editor Service
    • external.conceptbackend - Configuration details for how to connect to the Concept Backend service
    • binary.maxSizeMb.[im type] - Max size in mb for uploading a binary with specified Infomaker type, e.g. "x-im/image"
    • binary.binaryHeadRequestTimeoutMs - Timeout value in milliseconds for HEAD request made for checking binary availability

    Client configuration file

    {
  "newsItemTemplateId": "30eae1c0-c640-4053-b114-05c64e28bbe7",
  "language": "sv_SE",
  "labelLanguage": "sv",
  "forcePlaintextPaste": true,
  "plugins": [ ]
}
    • newsItemTemplateId - The article with this ID is used as template when a new article is served to the client.
    • language - Used for localization in Writer
    • labelLanguage - Used for internationalization in Writer labels
    • forcePlaintextPaste - true to force paste of plaintext, false to allow HTML paste
    • disableExternalSocialEmbeds - true to disable load of external script such as Facebook SDK and Twitter
    • facebookAppId - appId used to render social embed for Facebook
    • pluginLoadTimeout - timeout in milliseconds. The default value is 5000.
    • plugins - See Developer portal for more information about plugin development
