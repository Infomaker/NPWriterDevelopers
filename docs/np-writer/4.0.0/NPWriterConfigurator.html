NPWriterConfigurator

An NPWriterConfiguration instance object is passed to the plugin package configure() method. NPWriterConfiguration contains all the methods a plugin package needs to register the plugins features.

Constructor

new NPWriterConfigurator()

Source:
  • packages/npwriter/NPWriterConfigurator.js, line 36
Example
import './scss/index.scss'
import DevKitComponent from './DevKitComponent'

export default {
    name: 'npwriterdevkit', // TODO: Edit...
    id: 'se.infomaker.npwriterdevkit', // TODO: Edit...

    // The configure() is called by the writer when it wants the
    // plugin to initalize itself and its different parts.
    configure: function(config, pluginConfig) {

        // Add plugin to main sidebar (can be overriden in plugin config)
        config.addToSidebar('main', pluginConfig, DevKitComponent)

        // Add translations for the plugin
        config.addLabel('newsvalue', {
            en: 'Devkit plugin loaded',
            sv: 'Devkit plugin inläst'
        })
    }
}

addHook(type, implClass)

Source:
  • packages/npwriter/NPWriterConfigurator.js, line 448

Register a class with an execute method as a hook. All hook classes must implement an execute() method which returns a Promise.

Hook types

  1. before:validate, Before validation, can change NewsML through api calls.
  2. before:save, Before save, but after validation has passed. Changing NewsML not allowed.
  3. after:save, After save has succeeded.
Example
class MyHook {
   execute() {
      return new Promise((resolve, reject) => {
         /// Do Something
         resolve()
      })
   }
}
import {hook} from 'writer'
export default {
   name: 'myplugin',
   id: 'se.infomaker.myplugin',
   configure: function (config) {
      config.addHook(hook.AFTER_SAVE, NotifyComponent);
   }
}
Parameters:
Name Type Description
type string

Any hook type defined in {hook} from 'writer'
implClass class

Class implementing the hook execute method

addToSidebar(tabId, pluginConfig, component)

Source:
  • packages/npwriter/NPWriterConfigurator.js, line 266

Adds a component with pluginConfig to a new or existing tab in the right sidebar. Target tab is overridable from plugin config.

Examples

Used in a plugins *Package.js-file

config.addToSidebar(
     'main',         // Id/Name for selected tab
     pluginConfig,   //
     AboutComponent  // Component to render in tab
)

Tab override in config

{
    "id": "se.infomaker.genericplugin",
    "name": "genericplugin",
    "url": "http://localhost:5001/im-genericplugin.js",
    "style": "http://localhost:5001/im-genericplugin.css",
    "enabled": true,
    "mandatory": false,
    "tab": "Overriden Tab",
